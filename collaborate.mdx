## Development Setup

<AccordionGroup>
  <Accordion title="Core (Rust)" defaultOpen={false} icon="rust">
    ```sh
    # Clone the core repository
    git clone https://github.com/kontext21/k21.git
    cd k21

    # Install dependencies
    cargo build

    # Run tests
    cargo test
    ```
  </Accordion>

  <Accordion title="Node.js SDK" defaultOpen={false} icon="nodejs">
    ```sh
    # Clone the Node.js bindings
    git clone https://github.com/kontext21/k21-node.git
    cd k21-node

    # Install dependencies
    yarn install

    # Build the package
    yarn build

    # Run tests
    yarn test
    ```
  </Accordion>

  <Accordion title="Documentation" defaultOpen={false} icon="book">
    ```sh
    # Clone the docs repository
    git clone https://github.com/kontext21/docs.git
    cd docs

    # Install dependencies
    npm install

    # Start the development server
    npm run dev
    ```
  </Accordion>
</AccordionGroup>

## Development Guidelines

<AccordionGroup>
  <Accordion title="Code Style" defaultOpen={false} icon="code">
    * Follow language-specific best practices
    * Use meaningful variable and function names
    * Add comments for complex logic
    * Keep functions small and focused
  </Accordion>

  <Accordion title="Testing" defaultOpen={false} icon="check">
    * Write unit tests for new features
    * Add integration tests for complex flows
    * Ensure all tests pass before submitting
    * Maintain good test coverage
  </Accordion>

  <Accordion title="Documentation" defaultOpen={false} icon="book">
    * Update relevant documentation
    * Add examples for new features
    * Include JSDoc comments
    * Keep documentation up to date
  </Accordion>

  <Accordion title="Commit Messages" defaultOpen={false} icon="message">
    * Use clear and descriptive messages
    * Follow conventional commits format
    * Reference issues when relevant
    * Keep commits focused and atomic
  </Accordion>
</AccordionGroup>
